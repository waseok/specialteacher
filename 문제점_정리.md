# 시간표 자동생성 문제점 정리

작성일: 2026-05-05  
대상: 전담교사·특별실 시간표 자동생성 기능

## 1) 배정 정확도 문제

- 전담교사 시수 미달이 반복 발생함.
  - 예: 교사 주당 목표 20시간인데 자동배정은 10시간 수준으로 끝남.
- 영어 시수 미달이 핵심적으로 반복됨.
  - 예: 5·6학년 주당 3시간 설정인데 1~2시간만 배정되는 사례 다수.
- 같은 과목 일일 과배정 문제 발생.
  - 예: 과학이 하루 3시간 배정되는 비정상 케이스.
- 중간 공강이 생기거나, 다른 요일 4교시가 비어 있는데 6교시에 배치되는 등 배치 품질 저하.
- 한 요일에 여러 과목이 혼재되어 사용자가 기대한 “요일별 과목 집중”이 충분히 반영되지 않음.

## 2) 특별실 배정 문제

- 특별실 주당 배정횟수(`weeklyTarget`) 미충족이 반복됨.
  - 예: 주 1회 설정인데 실제 배정 0회.
- 한 교시에 여러 특별실이 동시에 얹히는 충돌 사례가 발생함.
  - 예: 운동장/도서실/음악실/컴퓨터실이 같은 교시에 중복되는 케이스.
- 특별실 우선순위 입력 시 중복 순위(1,1)가 발생해 우선순위 체계가 깨짐.
- 과목-특별실 연동 정책이 학교별 운영 현실과 불일치함.
  - 예: 놀이 과목이 다목적실로 고정되어 다른 학교 운영 방식 반영이 어려움.
- 과학실이 복수(예: 과학실 2종류)일 때 운영 규칙이 사용자에게 직관적으로 보이지 않음.

## 3) 자동생성 성능/안정성 문제

- 자동생성 시간이 과도하게 길어짐.
- `전담 + 특별실 추천 생성` 실행 시 진행이 멈춘 것처럼 보이는 상황 발생.
- 일부 케이스에서 로직 복잡도 증가로 체감상 “멈춤” 또는 “무한 대기”와 유사한 경험이 발생함.

## 4) 우선순위 정책 문제

- 사용자 기대 우선순위와 엔진 우선순위가 불일치했음.
  - 사용자 기대: **전담교사 설정 + 특별실 설정이 최우선**
  - 기존 동작: 과목 기본 우선순위/보정 로직이 더 강하게 작동하는 구간 존재.
- 교사 `weeklyTarget`, `allocSubject`, 특별실 `priority/weeklyTarget`이 항상 최우선으로 관철되지 않는 시점이 있었음.

## 5) 제약/경고 체계의 혼란

- 사용자 입장에서 이해하기 어려운 제약 메시지 발생.
  - 예: `교사일한도`, `교사중복` 등이 왜 막히는지 직관적으로 설명되지 않음.
- 제약이 과도하게 누적되면 배정률은 떨어지고, 문제 원인 파악은 더 어려워지는 구조가 노출됨.

## 6) UX(사용자 경험) 문제

- 자동생성 중 진행 상태(로딩/진행률/현재 단계)가 명확히 보이지 않아 불신을 유발함.
- “왜 특정 학급/과목이 미배정되었는지”를 즉시 확인할 수 있는 설명이 부족함.

## 7) 정리

현재까지의 핵심 문제는 아래 3가지로 요약됨.

1. **목표 충족 실패**: 교사 목표시수·과목 목표시수·특별실 목표횟수 미충족  
2. **배치 품질 저하**: 중간 공강, 6교시 과배치, 과목 일일 편중  
3. **신뢰성/가시성 부족**: 느린 실행, 멈춤 체감, 원인 설명 부족

---

## 부록: 사용자 관점 핵심 요구(우선순위)

1. 전담교사 설정(목표시수/주력과목) 최우선  
2. 특별실 설정(우선순위/주당횟수/연동) 최우선  
3. 그 다음 과목 균형 및 배치 품질(공강 최소화, 6교시 최소화, 요일 분산)
